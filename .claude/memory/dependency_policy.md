# Dependency Policy

## Version Pinning Rules
- **Python production**: exact versions in `requirements.lock` (generated by pip-compile)
- **Python dev**: exact versions in `requirements-dev.lock`
- **npm production**: exact versions (no caret/tilde) for framework deps (next, react)
- **npm dev**: caret ranges acceptable for dev tooling
- **Docker images**: pinned to minor version minimum (e.g., `python:3.11.15-slim`, `node:20.20.2-alpine`)
- **GitHub Actions**: pinned to commit SHA with version comment

## Adding a New Dependency
1. Add to `requirements.txt` (no version specifier — pip-compile resolves)
2. Run `pip-compile requirements.txt -o requirements.lock --strip-extras`
3. Run `pip-compile requirements-dev.txt -o requirements-dev.lock --strip-extras`
4. Verify with `pip-audit --strict --desc`
5. Commit all three files together

## Upgrade Process
1. Dependabot creates PRs for known CVEs automatically
2. For manual upgrades: edit `requirements.txt`, regenerate lock files
3. Run full test suite before merging
4. For Docker image upgrades: rebuild + scan with Trivy

## CVE Triage Criteria
- **CRITICAL/HIGH + RCE or data exfil**: Fix immediately, block merge
- **HIGH + DoS only**: Fix within 1 week (acceptable risk for localhost dev)
- **MODERATE**: Fix within 1 month or next dependency sweep
- **LOW + dev-only**: Track in Dependabot, fix opportunistically
